Bitfile Programming

Automates FPGA hardware detection, streamlining programming with a user-friendly interface

Bitfile Programming

This feature allows you to detect the connected FPGA board, core, vendor, bitfile. It streamlines the process of programming an FPGA device by automating the detection of necessary components and providing a user-friendly interface for initating programming operations.

1. FPGA Setup

Before proceeding, ensure that all required components (e.g., baremetal-neorv32-de2115-helloworld-example) and drivers for your FPGA board and JTAG cable are installed correctly. This includes drivers for the JTAG interface and any FPGA-specific toolchains such as Quartus or Vivado.

To ensure the JTAG connection is properly established, run the following command in the terminal. This will provide detailed information about the detected JTAG devices and their connection status. Ensure that you are using the correct tools depending on the FPGA vendor (Intel or Xilinx):

2. Open Bitfile Programming View

  • Launch the IDE.
  • Navigate to View > Bitfile Programming to open the Bitfile Programming view.

3. Bitfile Programming Interface

The Bitfile Programming dialog consists of the following components:

  • FPGA vendor tabs: Switch between Intel FPGA and Xilinx FPGA
  • Bitfile selection: Choose between:
    • Default Bitfile: Automatically selects the default bitfile associates with the detected FPGA board.
    • Custom Bitfile: Allows manual selection of a bitfile from the filesystem.
  • Board information: Display detected board details, including:
    • Board Name
    • Core
    • Vendor
    • Description
  • JTAG cable information: Automatically displays connected JTAG cable details
  • Programmer Path: Specify the path to the programmer tool (e.g., Quartus Programmer for Intel).

You can switch between Default Bitfile and Custom Bitfile using the tabs in the interface. Each tab updates the bitfile selection options accordingly.

Configure the Programmer Tool

  • In the Programmer field within the dialog, specify the path to the Quartus Programmer or Vivado Programmer tool (e.g., /intelFPGA_lite/23.1std/quartus/bin/), if not already configured.

  • Once the programmer path is set, the feature will automatically detect the FPGA board, core, vendor, and JTAG cable information.

Note

The Programmer path must be configured before you can detect the JTAG cable and proceed with programming. Without the correct configuration, the detection and programming process may fail.

4. Program the FPGA

  • Click the Run button in the GUI.

  • The IDE will execute the appropriate programming command using the configured tool path.

  • The programming status will be displayed in the Output View.

5. Check the Results

  • After pressing the Run button, a progress bar will appear in the bottom-right corner. If successful, a confirmation message will also appear in the same area.

  • If an error occurs, detailed error information will be provided in the Output View for troubleshooting.
