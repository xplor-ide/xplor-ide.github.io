General IDE

This page describes a general overview of the XploR Eos IDE.

User Interface Introduction

Welcome to XploR Eos IDE, a powerful and extensible Integrated Development Environment (IDE) designed for developers who write embedded software. Based on the Theia IDE open-source software framework, XploR Eos IDE provides a seamless and customized environment tailored for both beginners and experienced developers.

XploR Eos IDE uses a standard IDE layout, with a project and file explorer on the left and an editor on the right, designed to maximize space for essential development tasks. The key sections of XploR Eos IDE include:

  • Editor: The main area for writing and editing code files.
  • Activity Bar: Provides various views for different tasks, such as the Explorer for project management or the Debug view for debugging sessions.
  • Status Bar: Displays useful information about the open files and the current workspace.
  • Side Bar: Contains different views like the Explorer or debugging tools, based on the selected activity.
  • Panels: Positioned below the editor, panels show information such as build status, debugger output, and terminal access. These panels can be moved to the right or repositioned elsewhere in the main window, according to user preferences.

Activity Bars

The left Activity Bar feature six main views:

  • Explorer: Navigate, open, and manage files and folders within your project. For additional details about the Explorer, click here.
  • Search: Look for files in your current workspace.
  • Run and debug: Execute your program in debugging mode while monitoring threads, the call stack, variables, watched expressions, and breakpoints. More information about the debugger can be found here
  • Source control: Monitor changes in your repository, including modified files, staged changes, and merge changes. Here, you can stage, unstage, and commit your files. The source control icon display the number of changes currently present in your repository.
  • Extensions: Add tools to enhance your development process by installing extensions. You can find additional details about the extensions
  • XploR Labs: Connect to cloud from local IDE to perform remote workspace and remote debug features

At the bottom of the left Activity Bar, you’ll find a Settings icon, which provides access to general settings, keyboard shortcuts, and options to modify color themes and file icons.

On the right side, the Activity Bar includes an Outline for symbols in your file, facilitating easy navigation. As you open new views, they will automatically populate your Activity Bars. By right- clicking on a view icon, you can choose to close it, pin it to the Activity Bar, or collapse the opened view. You can also drag views between the Activity Bars and the Panel.

Extensions

You can search for and install extensions through the Extensions icon on the Activity Bar. For instance, you might search for the Code Spell Checker extension.

This tool helps you identify and correct spelling errors in your code and documentation by highlighting misspelled words.

To remove an extension, go to the Extensions icon, find it either by searching or under the installed tab, and select uninstall. A wide range of extensions is available for different purposes, which you can explore at https://open-vsx.org/. You can always switch to another extension as needed.

Menus

Menus play a key role in providing a user-friendly interface for accessing various commands and features. Here’s an overview of how menus function in XploR Eos IDE:

  • File: This menu provides options related to workspace and project management. Common actions include:

    In addition to these, the File menu includes several other options for managing files and workspaces, enabling more granular control over your development environment.

  • Edit: Provides standard editing commands.

  • Selection: Tools for managing selections in the editor.

  • View: Controls the visibility of different panels and views.

Editors

The XploR Eos IDE editor is built on the VS Code editor, sharing many similarities. For more information, please refer to the Basic Editing. Keep in mind that while the core functionalities of the editors are very similar, there are some differences between the two environments.

Status Bar

The Status Bar displays information about your files. You can interact with this information to modify settings, check for errors and warnings, view notifications, and toggle the Panel. Through the Status Bar, you can adjust the end-of-line sequence, encoding, indentation, and language mode settings. By clicking on the line number, you can quickly navigate to a specific line in your file.

Terminal

Integrated Terminal

Create a new full featured integrated terminal that starts at the root of your workspace.

An external can be opened by using Ctrl+Shift+`. The shell settings can be modified in File -> Preferences -> Settings -> Terminal.

You can split the terminal by pressing the Split Terminal icon in the top right corner of the terminal or move terminal to the secondary view.

SSH Terminal

Provides a user-friendly dialog interface that enables users to securely connect to remote machines using the SSH protocol.

To open this terminal, click on the Terminal and select New SSH Terminal.

A dialog will appear, prompting you to enter the necessary connection details.

In the dialog, you will find fields to input the following information:

  • Host: Enter the IP address or hostname of the remote machine you wish to connect to.
  • Username: Provide your username for authentication on the remote machine.
  • Password: Input your password (masked for security) associated with the specified username. If you prefer not to use a password for SSH authentication, you can utilize a private key. Here’s how you can do it:
    • Generate an SSH key pair: If you haven’t already generated an SSH key pair, you can create one using the following command in your terminal: ssh-keygen -t rsa -b 2048. This will create a public key (id_rsa.pub) and a private key (id_rsa) in your ~/.ssh directory.
    • Copy the contents of your public key (id_rsa.pub) to the remote server’s ~/.ssh/authorized_keys file. You can use the following command: ssh-copy-id username@hostname
    • Once the public key is added to the remote server, you can connect to it without entering a password. In the SSH Terminal dialog, simply leave the password field blank.
  • Port: Specify the port number for the SSH connection (default is 22).

Click OK button to establish the SSH connection.

When the connection to the remote server is successful, a message will be displayed in the Panel.

Serial Terminal

The Serial Terminal feature allows users to establish a serial connection on a specified port. Follow these steps to set up and verify the serial communication:

To open the Serial Terminal, in the Terminal View of your development environment, select New Serial Terminal to access the serial communication configuration.

On Linux, choose the corresponding /dev/ttyUSBx port. You can list available ports using ls /dev/ttyUSB* in the terminal.

Make sure that the serial connection settings are configured correctly. Typical parameters include:

  • Baud Rate: e.g., 115200
  • Data Bits: 8
  • Stop Bits: 1
  • Parity: None

After configuring, connect to the serial terminal by click the Connect button. If the setup is correct, there is a terminal with the title being the name of the port opened and the output from that port.
