Managing Projects

Projects Overview

This section provides an overview of workspaces and projects and some features support users working more conveniently.

Workspaces

A “workspace” is the collection of one or more folders that are opened in the File Navigator. In most cases, a single folder is opened in the workspace. However, it is possible to have multiple folders open simultaneously, known as a multi-root workspace environment.

  • Configure settings that only apply to a specific folder or folders but not others
  • Persist task and debugger launch configurations that are only valid in the context of that workspace
  • Store and restore UI state associated with that workspace (for example, the files that are opened)
  • Selectively enable or disable extensions only for that workspace

Single-Folder Workspace

You can open a folder the following way:

  • File → Add Folder to Workspace… and specify the folder to switch to.

  • Click button Add Folder To Workspace and specify the folder to add.

Multi-Root Workspace

Multi-root workspaces are workspaces with multiple distinct folders associated with it. All the folders appear in the File Navigator for that workspace. Instead of opening a folder as workspace, they use a *.xplor-workspace JSON file that lists the folders of the workspace.

Add Folder to Workspace

  • New folders can be added to a workspace by selecting the More Actions… → Add Folder to Workspace… option in the File Navigator toolbar.

  • You can also add a new folder to a workspace by going to File → Add Folder to Workspace….

After a folder is specified, a prompt will appear asking if you wish to save the workspace to a file. It is not required to save the workspace to a file. However, it may be useful if you are working with multiple sets of multi-root workspaces.

Note: You will still be able to save the untitled workspace later if you want to. Otherwise, it will only remain accessible till you end the current session of XploR IDE.

Once the steps are completed, both folders will now be available in the File Navigator for the current workspace:

Saving Workspace

The current workspace information can be saved to a file and re-imported:

  • To save workspace settings: File → Save Workspace As….

After saving workspace, workspace name in File Navigator will be updated.

  • To import workspace settings: File → Open Workspace From File….

The file is a json file with a *.xplor-workspace file extension. It will contain paths to any open folders in the File Navigator and any custom workspace settings.

Opening An Exist Workspace

A *.xplor-workspace json file is used to represent a multi-root workspace, and store custom settings for workspaces. The schema is as following:

{
   "folders": [
      {
         "path": "<path-to-folder-1>/<folder-name-1>"
      },
      {
         "path": "<path-to-folder-2>/<folder-name-2>"
      }
   ],
   "settings": {
      "<setting-key-1>": "<setting-value-1>",
      "<setting-key-2>": "<setting-value-2>"
   }
}

When you save any workspace to a file, it is saved as a *.xplor-workspace file. The folder paths are automatically generated so they are the correct location. *.xplor-workspace file supports both absolute and relative file paths.

Workspace Settings

Workspace settings enable you to configure settings in the context of the workspace you have opened and always override global user settings.

Projects

A project stores all the information needed to build an individual program or library, in case a project be created from our template, including:

  • Filenames and paths of source code and object libraries.

  • Debug folder: Makefile to build project.

  • .xplor folder: Include file dependencies.

  • .theia folder: Include build option, define tasks to build and declare debug option.

  • package.json: Include project info (template name, dependencies, language,…)

  • Projects typically contain files and folders. Similar to the workspace, a project maps to a physical folder in the file system. When a new project is created, its default location is in a subfolder (of the project name) within the workspace folder. However you can also choose a folder outside the workspace.

Once the project is created, a reference to it will be made in the workspace and the project will be visible and available for use from the File Navigator. Each project in the workspace must have a unique name.

On the other hand, projects created or added to the workspace will also be added to the Project Management Box. Instructions for using this feature can be found here.

Xplorer

The Xplorer can be opened in the Side Bar by selecting the Xplorer icon in the side Activity Bar. Xplorer has 2 main feature:

  • Display the workspace folder and all associated projects in File Navigator.

  • Display information and support from create to build project in Project Management Box.

The primary function of the File Navigator is to display the workspace folder and all associated projects. The project displayed is mostly a representation of the file system of the project folder. When you create a subfolder and move files to that subfolder from within the File Navigator, the actual filesystem is being altered. Similarly, changes made to the filesystem will be reflected in the File Navigator.

Note that not all files that appear in the view will exist in the filesystem and vice versa. Linked files will appear in the view but because they are references and not actual copies, they will not appear in the actual filesystem. Any project metadata files/folder with the project folder (typically preceded with a “.”) will also be hidden.

The screenshot below shows the contents of the ‘helloworld_cpp_project’ project in the Explorer and how it maps to the physical project folder in the workspace in Windows Explorer. Note how the .theia, .xplor and .settings folder shown in physical project folder hidden in the File Navigator.

Otherwise the rest of the folders and files are structured the same in the File Navigator and physical project folder.

The main feature of Project Management Box is display and from create to build project active project. The information include:

  • Active Project: The project is focus by user.
  • Template: In case project has package.json file contain information and package.json file has name property.
  • Build: All build profiles is configured in .theia/tasks.json.
  • Debug: All Debug configuration in .theia/launch/json

In case XploRer widget has multiple project, there are two ways to change Active Project:

  • Choose from pulldown.

  • Click in any file/folder of project you need to be Active Project.

And Project Management Box has 4 main function:

  • Create Project: Create project with installed templates in local machine.
  • Install Dependencies: Find and install any published templates.
  • Build: Build active project by chosen profiles.
  • Debug: Start debug by chosen debug session option.

How to use the features as well as the entire flow from template installation to build is presented at Working with Projects and instructions for configuring the build profile are presented 5.2 Building Projects.

Working with Projects

In this section cover the flow from install template to build it with any option you want.

Install a Template

You can find all published templates by clicking the install dependencies button.

This Dialog show all information included in package.json file of each template:

  • Name: Template’s name.

  • Description: Some basic descriptions about the purpose of the template (hello world), project type (Baremetal, FreeRTOS,… ), target type (QEMU or FPGA).

  • Author: vendor that provide the template for us.

  • Installed: The template is installed on local machine or not.

  • Latest: Latest version of template in registry server.

After open installation dialog, you can choose your template currently or use filter to find the template you need.

We have some option to filter published template, first is pulldown filter option:

  • Language: Language is used into template (C, C++).

  • Type: Template project type (Baremetal, FreeRTOS,…).

  • Target: Target of template project (QEMU, FPGA).

Besides, our ide also supports searching by user’s optional keyword. Users can enter one or more keywords in the Other option box, each keyword separated by a comma. This feature do not check match case, user can fill it with upper/lower case, ide will return all template match with given keywords.

The keywords can be apart of name, author or description. Therefore, the greater the number of keywords, the more accurate the returned results will be.

After selecting the filter options and filling in the desired keywords, press enter or click the apply filter button to filter out related templates.

Check in the box of template you need to install and click button install to start install.

After install template completely, you can find it into create project dialog.

If you want to clear all conditions, click the reset filter button and start searching template from begin.

Create New Projects

Xplor Eos IDE support to create project from installed template in UI. Firstly, click button Create Project to open Create Project Dialog.

This dialog can help you create project by:

  • Project name: Name of your project, our IDE will help you to validate this name
  • Template: Installed template in your local machine
  • Description: Some basic descriptions about the purpose of the template (hello world), project type (Baremetal, FreeRTOS,… ), target type (QEMU or FPGA).
  • Location: Saving location of project (in default, it point to ~/.xplor/xplor-workspace)
  • Install New Template button: Help to find and download any publish template you need.

After open Create Project dialog, you can create project by choose the template that installed:

And saving location :

Create Project Dialog support to validate your project name in 2 case and you can not create project:

  • Check syntax: Special letter,…

  • Check duplicate: Warning if saving location has any folders have same name with your new project name.

The warning will be removed as soon as the project name is fixed, now you can click button Create or press enter to create project.

When new project is create successfully, it will add to recent workspace and you can see it in File navigator and Project Management Box.

In case your local machine do not have template you want, click Install New Template button to open Install Dependencies Dialog.

Instructions for setting templates, please see section 5.1.2.1 Install a Template

Import Projects

XploR IDE use workspace to manage project, it allow to import exist project into current workspace.

First of all, click More Action button in File Navigator and click Add Folder to Workspace.

After that, choose import project and click button Open to add it into current workspace.

Now, the chosen project is added into workspace. You can see it into File navigator and Project Management Box.

Currently this feature is not available on XploR IDE, we will update it soon.

Delete Projects

Xplor IDE just allow remove project from current workspace.

  1. Right click in project
  2. Click remove folder from Workspace
