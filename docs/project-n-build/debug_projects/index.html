Debugging Projects

This section explains how to debug the projects

Xplor IDE support for debug project currently in Xplorer, this section will guide you on how to start debug in Xplorer, for more complete information about debugging features, debugging steps as well as debug configuration, please see in Debug Project.

Launch Configurations

A launch configuration (or debug configuration) is used to set up the debugger when starting a debug session. The IDE typically auto-generates this information and stores it in a launch.json file within the .theia folder of your workspace.

Launch configurations in XploR Eos IDE function similarly to those in VS Code. For more details, refer to the VS Code documentation for Launch Configurations.

When setting up a launch configuration in XploR Eos IDE, the following options are available:

  • cwd: Set the current working directory (usually your workspace folder).
  • executable: Specify the path to your application’s binary.
  • name: Give your debug configuration a unique name.
  • type: Choose the debug adapter type (xplor-debug or your external one)
  • servertype: Select the GDB Server type (options: openocd, qemu, external).
  • configFiles: Define OpenOCD GDB Server configuration files for debugging (OpenOCD -f option).
  • searchDir: Specify directories for OpenOCD to search for config files and scripts (-s option). Defaults to the configured cwd if not set.
  • runToEntryPoint: Enable this to run the debugger until the start of a specific function (applies to Launch/Restart/Reset, ignored for Attach).
  • postLaunchCommands: Add custom GDB commands to execute after the main launch sequence.
  • serverPath: Override the GDB Server path for this configuration (full path or executable name if in PATH).
  • gdbPath: Override the GDB path for this configuration (full path or executable name if in PATH). Ensure other toolchain executables with the configured prefix are available.
  • gdbPort: Override the GDB port for this configuration.

For all available options for a launch configuration in XploR Eos IDE, please refer to Debug Configuration.

When using the QEMU servertype, you’ll need to configure the following additional options:

  • machine: Select the machine type (specific to QEMU).
  • cpu: Choose the CPU type (specific to QEMU).
  • serverArgs: Add any extra arguments to pass to the GDB Server command line.

Add a New Launch Configuration

There are multiple ways to add or create a new debug configuration in XploR Eos IDE:

  • From Xplorer View: Open the Debug dropdown, select Add Configuration, and ensure you choose the Active Project you want;

  • From Debug View: Click the Add Configuration option in the dropdown, then select the project for which you want to create a launch configuration:

Start a Debug Session

There are several ways to add or create a new debug configuration in XploR Eos IDE:

  • From Xplorer View:

    • Open the Debug dropdown, select the desired debug configuration:

    • Click the Debug button.

  • From Debug View:

    • Select the debug configuration:

    • Click the Start Debugging button.

Once the debug session starts, you’ll be redirected to the Debug View, where you can continue your work seamlessly.
