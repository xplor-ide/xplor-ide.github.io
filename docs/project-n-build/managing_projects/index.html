Managing Projects

This section provides an overview of workspaces and projects.

Concepts

This section provides an overview of workspaces and projects and some features support users working more conveniently.

  • Workspaces: Includes the created projects and additional folders, providing a unified and organized environment for efficient development.
  • Projects: Initialized from a template, contains all necessary information and resources to develop a specific program.
  • Xplorer: Navigates, opens, and manages files and folders within your project.

Workspaces

A “workspace” consists of one or more folders opened within the File Navigator. It uses a *.xplor-workspace JSON file to define and list the folders included in itself.

When the IDE is launched for the first time. It have a default workspace with name xplor-workspace. The *.xplor-workspace JSON config file can be automotive generated with folder xplor-workspace. Any project created by the IDE will be set the default location to this folder.

In this setup, you have the flexibility to:

  • Apply settings to specific folders without impacting others.
  • Save task and debugger launch configurations that are specific to that workspace.
  • Store and recover the UI state, such as which files were open, for that workspace.
  • Enable or disable extensions exclusively for that workspace.

Add Folder to Workspace

  • To add a new folder to the workspace, you can use the More Actions… → Add Folder to Workspace… option in the File Navigator toolbar.

  • Alternatively, you can add a folder by navigating to File → Add Folder to Workspace….

Once the folder is added, both folders will be visible and accessible in the File Navigator for the current workspace:

Saving Workspace

You can save the current workspace information to a file and later re-import it for use:

  • To save workspace settings: File → Save Workspace As….

Once the workspace is saved, the workspace name in the File Navigator will be updated to reflect the new file.

Opening An Exist Workspace

A *.xplor-workspace JSON file is used to define and represent a workspace, as well as to store custom settings specific to that workspace. The structure of the JSON file follows this schema:

{
   "folders": [
      {
         "path": "<path-to-folder-1>/<folder-name-1>"
      },
      {
         "path": "<path-to-folder-2>/<folder-name-2>"
      }
   ],
   "settings": {
      "<setting-key-1>": "<setting-value-1>",
      "<setting-key-2>": "<setting-value-2>"
   }
}

When you save a workspace to a file, it is stored as a *.xplor-workspace file. The folder paths within the file are automatically generated to ensure they point to the correct locations. The *.xplor-workspace file supports both absolute and relative file paths, providing flexibility in how the workspace is configured and shared.

The workspace can be opened into the IDE in 2 ways:

  • Open by File: Open by file is a way to select an exact *.xplor-workspace file containing information about the workspace that needs to be opened.

To import workspace settings: File → Open Workspace From File….

The workspace file is a JSON file with a *.xplor-workspace extension. It includes the paths to all folders currently open in the File Navigator and any custom workspace settings that have been configured.

  • Open by Folder: Open by folder is a way to select a folder and the IDE will create an empty workspace with a *.xplor-workspace file.

To open a workspace by folder: File → Open Workspace From Folder…

The created workspace has the same name as the folder used to open it.

Workspace Settings

Workspace settings allow you to customize configurations specifically for the workspace you are currently using. These settings take precedence over global user settings, ensuring that the workspace operates according to your defined preferences, regardless of the broader user configuration.

Projects

A project stores all the information needed to build an individual program or library, in case a project is created from our template, including:

  • Filenames and paths of source code and object libraries.
  • .xplor folder: Contains symbolic links to all the project dependencies.
  • .theia folder: Include build option, define tasks to build, and declare debug option.
  • package.json: Include project info (template name, dependencies, language,…)

Projects generally consist of files and folders. Additionally, a project corresponds to a physical directory in the file system. When a new project is created, it is placed by default in a subfolder (named after the project) within the workspace directory. However, you also have the option to select a folder outside the workspace for the project.

Once the project is created, it is linked to the workspace and becomes accessible through the Project Management section. Every project within the workspace must have a distinct name.

On the other hand, projects created or added to the workspace will also be added to the Project Management section. Instructions for using this feature can be found in Working with Projects.

Xplorer

The Xplorer can be opened in the Side Bar by selecting the Xplorer icon in the side Activity Bar. Xplorer has 2 main features:

  • Display information and support creating, building, and debugging projects quickly in the Project Management section.

  • Display the workspace folder and all associated projects in File Navigator.

  • Display the editing files and allow saving all changes in Open Editors.

The File Navigator is designed to display the workspace and all its linked projects. It essentially reflects the structure of the project folder as it exists in the file system. Actions like creating subfolders or moving files within the File Navigator directly modify the actual file system. Similarly, any changes made to the file system will automatically update in the File Navigator.

However, there are some differences between what is shown in the File Navigator and what exists in the file system. For instance, linked files appear in the File Navigator as references but do not physically exist in the file system.

The accompanying screenshot demonstrates the contents of the helloworld_cpp_project in the File Navigator and how it aligns with the physical project folder in Windows Explorer. As shown, folders like .theia and .xplor which are present in the physical folder, are not visible in the File Navigator.

Otherwise, the folder and file structures in the File Navigator and the physical project folder are identical.

The main feature of the Project Management section is to display information and allow users to create, build, and debug the active project. The information includes:

  • Active Project: The project is focused by the user.
  • Template: A project template’s name, usually indicates its application.
  • Build: All build profiles are configured in .theia/tasks.json. More details information in Building Projects.
  • Debug: All Debug configurations are in .theia/launch.json. More details information in Debugging Projects.

In case the Xplorer view has multiple projects, there are two ways to change Active Project:

  • Choose from pulldown.

  • Click in any file/folder of the project you need to be Active Project.

The Project Management section has 4 main functions:

  • Create Project: Create a project with installed templates on the local machine.
  • Install Templates: Find and install any published templates.
  • Build: Build active project by chosen profiles.
  • Debug: Start debug by choosing a debug session option.

How to use the features as well as the entire flow from template installation to build is presented in Working with Projects and instructions for configuring the build profile are presented in Building Projects.

Working with Projects

A project template is a project prototype, encompassing source code, build and launch configurations, and dependencies on external tools (such as toolchains, OpenOCD, etc.). It ensures that the application can be successfully built and debugged, providing a ready-to-use foundation for development. This section covers the working flow with projects as installing a template, creating a new project, importing, renaming, or deleting the projects.

Installing Packages and Project Templates

The Package Manager allows users to browse, search, and install project templates in the XploR Eos IDE. This guide explains how to access the Package Manager and install a package or project template.

Accessing the Package Manager

To install a package or project template, follow these steps:

  1. Open XploR Eos IDE.
  2. Click on the Install Templates button in the Xplorer section.
  3. The Package Manager dialog will appear, displaying a list of available packages.

Alternatively, you can also open the dialog by clicking on the View menu and selecting Package Manager.

Package Manager interface overview

Once inside the Package Manager, you will see the following elements:

  • Search Bar: Allow users to search for specific packages by name or description.
  • Filter Dropdown: Allow filtering installed packages based on categories.
  • Package List: Display available packages with the following columns:
    • Name: The package’s unique identifier.
    • Author: The organization or individual supplying the package.
    • Description: A short summary of the package’s purpose
    • Version: The package’s version number.
  • Selection Checkbox: User can select one or multiple packages for installation.
  • Action Buttons:
    • Install: Install the selected packages.
    • Close: Close the Package Manager dialog.
    • Installed Packages: Open the Installed Packages dialog to manage already installed packages.

Installing a package or project template

  • Step 1: Search or browse for a package

    • Use the search bar to filter packages by keyword.
    • Browse the list to find the desired package.

  • Step 2: Select the package

    • Check the box next to the package name to mark it for installation.
    • Optionally, select a specific version from the dropdown menu.

  • Step 3: Confirm installation

    A confirmation dialog will appear, displaying:

    • Packages that are already installed.
    • Packages that will be newly installed.
    • Checkbox for reinstallation.

  • Step 4: Install the package
    • Click the Install button to begin the installation process.
    • A progress indicator may appear, showing the status of the installation.
    • Once completed, a success message will appear in the bottom-right corner, confirming the installation.
Managing Installed Packages

To view and manage installed packages, click the Installed Packages button in the Package Manager. This will open the Installed Packages dialog, which provides an overview of all currently installed packages.

Installed Packages interface overview:

  • Search Bar: Helps users quickly find installed packages.
  • Filter Dropdown: Allows filtering installed packages based on categories.
  • Installed Template List: Displays installed packages.
  • Action Buttons:
    • Uninstall: Removes the selected packages.
    • Close: Closes the Installed Packages dialog.

How to uninstall a package
  • Open the Package Manager and click Installed Packages.
  • Use the Search Bar or Filter Dropdown to locate a specific package.
  • Select the checkbox next to the package(s) you want to uninstall.
  • Click the Uninstall button to remove the selected package(s).
  • Confirm the action if prompted.

Warning

The Package Manager allows users to install various packages, including project templates and system dependencies. However, uninstalling certain packages—especially non-template ones—may disrupt system functionality or cause errors. Proceed with caution when removing installed packages.

Create a New Project

Xplor Eos IDE support to create projects from installed templates in UI. Firstly, click button Create Project to open the Create a new project dialog.

This dialog can help you create a project by:

  • Project name: Name of your project, the IDE will help you to validate this name.
  • Template: Installed template in your local machine
  • Description: Some basic descriptions about the purpose of the template (hello world), project type (Baremetal, FreeRTOS,… ), target type (QEMU or FPGA).
  • Location: Saving location of the project (in default, it points to ~/.xplor/xplor-workspace)
  • Install New Template button: Help to find and download any published template you need.

The dialog also supports to validate your project name in 2 cases and you can not create a project if the project name:

  • has special letter like: !, ~, `, etc.
  • has the same name as an existing folder in the location where you save the project.

When a new project is created successfully, it will be added to the recent workspace and you can see it in the File Navigator and Project Management section.

In case your local machine does not have the template you want, click the Install New Template button to open the Package Manager dialog.

Instructions for setting templates, please see section Install a Template.

Import Projects

XploR Eos IDE uses workspace to manage projects, it allows importing any folders present on the local machine, managing them as projects within the workspace they are imported into.

First of all, click the More Actions… button in the File Navigator and click Add Folder to Workspace….

Now the chosen project is added to the workspace. You can see it in the File Navigator and Project Management section.

Note

  • Build and Debug fields scan the tasks.json and launch.json files in the .theia folder to display information.
  • Missing this folder will leave these two fields blank. For details about the two files tasks.json and launch.json,
  • See details in Build Configurations and Debug Configurations.

Remove Projects from the workspace

Xplor Eos IDE just allows removing projects from the current workspace.

  1. Right-click on the project
  2. Click Remove folder from Workspace

Note

Currently, XploR Eos IDE does not allow deleting projects using the IDE. Please delete the project manually outside the IDE.
